Тип грамматики - в этом блоке выбирается необходимый тип генерируемой грамматики.В случае автоматной грамматики в появившемся блоке "тип автоматной грамматики" дополнительно можно выбрать левой или правой будет сгенерированная грамматика. Если же это неважно, оставьте этот блок без изменений.

Нетерминалы - здесь указывается количество нетерминалов, которые будут использоваться при генерации. Изначально уже заданно некоторое число, которое при необходимости можно изменить. Галочка "Установить по умолчанию" устанавливает количество нетерминалов на 26 (полный латинский алфавит).

Правила - в этом блоке задается количество замен для каждого нетерминала, а также длинна каждой замены. Изначально заданное значение можно изменять. Доступно три режима регулирования количества правил. 
Первый - "Установить по умолчанию": устанавилвает количество правил на 5, а длинну - на 4. 
Второй - "Конкретное значение". В этом режиме возможно задать только точные значения для каждого из параметров. 
Третий - "Диапазон значений". Каждый параметр можно задать в диапазоне от числа А, до числа Б. Какое конкретное будет выбрано, решает сама программа.

Сгенерировать - при нажатии на эту кнопку будет сгенерированна грамматика с указанными ранее параметрами. Для корректной генерации должны быть заданы все значения из пердыдущих блоков. После генерации полученная грамматика будет выведена в текстовое окно под самой кнопкой. При этом у вас появится возможность сохранить эту грамматику нажатием кнопки "Сохранить в файл". Не забудьте указать путь к файлу в появившемся окне.

Грамматика генерируется в виде "S-a|b|c", где S - заменяемый нетерминал (в случае контекстно-зависимой - нетерминал с контекстом), a..c - замены. Последняя замена всегда генерируется из терминальных символов (в случае контекстно-зависимой к ней еще добавляется контекст).

Особенности генерации:
1. Последняя замена всегда терминальна.
2. В случае, когда каждому нетерминалу должна сопоставлятся одна замена, то для всех нетерминалов, кроме последнего генерируется одно нетерминальное правило, а для последнего - нетерминальное и терминальное. Это сделано для сохранения полноты и конечности грамматики.
3. В случае, когда количество нетерминалов равно единице, генерируются только терминальные замены.
